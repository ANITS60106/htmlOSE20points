Git rebase - практически похож с git merge, но в семье есть подкидыши, но мы черт возьми аристократы, поэтому скажу официально, то: это способ поддерживать порядок в истории изменений и делать её более логичной. Перемещение изменений: rebase берет изменения из одной ветки и применяет их на другой ветке, будто бы в одно время. Чистая история: Это делает историю проекта более понятной, такой что даже такой идиот как я понял, так как все изменения идут последовательно, без разветвлений, которые могут возникнуть при слиянии. Конфликты(срачи): Иногда при использовании rebase могут возникнуть конфликты, которые нужно будет разрешить, как и при слиянии. Отличия от гит merge: merge сохраняет историю с ветвлениями, а rebase создаёт более линейную и чистую историю изменений.

Git fetch - бл*ть те же братья близнецы, но на этот раз с git pull. Аристократичными словами: это способ проверить, что произошло в удалённом репозитории, не меняя при этом твою локальную историю. После этого ты можешь решить, например, выполнить git merge или git rebase, чтобы объединить изменения. Загрузка изменений: Когда ты выполняешь git fetch, гит скачивает все новые коммиты и данные из удалённой ветки, но не вносит их в твою текущую ветку. Обновление информации: После выполнения git fetch у тебя будут все последние изменения, но ты останешься на своём текущем комите, то есть там где ты сейчас. Это позволяет посмотреть, что нового добавилось, прежде чем решать, что с этим дерьмои делать и стоит ли портить жизнь напарнику. Отличия от git pull: fetch просто обновляет информацию о удалённых изменениях, а pull обновляет и объединяет эти изменения с локальными

GGit workflow — это способ организации работы с Git. Вот основные шаги: Создание ветки: Я понял, что для каждой новой функции или исправления создаётся отдельная ветка. Работа над изменениями: тупо комиты. Объединение: Когда работа завершена, объединяем свою ветку с основной . Разрешение конфликтов: Если возникают конфликты, их нужно разрешить перед объединением. Такой подход помогает поддерживать порядок и упрощает совместную работу над проектом.
